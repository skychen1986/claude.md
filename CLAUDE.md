# LLM Wiki Schema — 个人知识库

本文件定义 LLM 如何维护这个 Obsidian vault。基于 Karpathy LLM Knowledge Bases + Codex 四层自生长循环。

## 四层知识循环

```
A-原始素材 (raw/)  →  B-提炼知识 (方法论/)  →  C-技能库 (技能/)  →  D-产出 (项目/ + 日志/)
       ↑                                                                                      ↓
       └────────────────────────── 回流：产出定期复盘后重新进入循环 ←──────────────────────────┘

- **A层 raw/**：原始素材，只读，绝不修改
- **B层 方法论/**：跨项目方法论、铁律、框架——从A层和对话中蒸馏的通用知识
- **C层 技能/**：可执行技能——从B层封装的操作化提示词/规则集，直接调就用
- **D层 项目/ + 日志/**：项目产出和每日记录，定期复盘后回流B层

## Directory Structure

```
vault/
├── raw/               # A层 · 原始素材 — 只读
│   ├── 原稿/
│   └── 参考/
├── B-提炼知识/         # B层 · 跨项目方法论、铁律、框架
├── C-技能库/           # C层 · 可执行技能（提示词/规则集）
├── wiki/              # 结构化知识索引
│   ├── index.md
│   ├── 笔记索引.md
│   └── 开发日志/
├── projects/          # D层 · 项目知识库
│   ├── 项目A/
│   ├── 项目B/
│   └── ...
├── daily/             # D层 · 每日工作日志
├── 对话记录/           # 每日对话日志
├── 废弃内容/           # 溯源层 · 只增不删不改，已删除/替换文件的永久存档
├── .daily-tasks.json
└── .daily-schedule.json
```

## Operations

### Ingest（A层 → B层）
1. 新素材进入 `raw/`（用户拖入或 Web Clipper）
2. LLM 读取，提炼可复用方法论进 B层
3. 对话中产生的决策、框架、规则→自动写入B层
4. 更新 `wiki/笔记索引.md`

### Distill（B层 → C层）
1. B层知识经实践验证后→封装为C层可执行技能
2. 触发条件：同一方法论被使用3次以上→封装Skill
3. 技能格式：触发条件+输入+执行规则+输出+禁用项
4. 每周复盘时检查B层是否有待封装的知识

### Apply（C层 → D层）
1. 执行任务时先查C层是否有匹配技能
2. 调用技能→产出进入D层
3. 产出中发现的改进→反馈回B层

### Circulate（D层 → A层）
1. 每周复盘：本周产出中有哪些新认知？
2. 蒸馏后写入B层
3. 更新对应C层技能
4. 产出中的失败/踩坑也回流——不只看成功

### Lint（健康检查）
1. 定期检查四层之间的链接
2. B层知识是否有对应的C层技能？如果没有→判断是否需要封装
3. C层技能是否被实际使用？如果长期未用→标注或删除
4. 检测死链、孤立页面、过期信息

## Rules

1. **绝不编辑 `raw/`** — 原始素材永远只读
2. 每次修改/新增 MD 文件后更新 `wiki/笔记索引.md`，对话结尾修正日期
3. 所有交叉引用使用 `[[wikilinks]]` 语法
4. 新 wiki/B层/C层 页面开头标注来源
5. 同方法论使用3次→自动封装C层技能
6. 每周复盘检查循环是否在转（A有新料？B有更新？C有封装？D回流了？）
7. 知识命名用中文，文件名简洁明确
8. **绝不直接删除 vault 内任何文件** — 需要删除/替换的内容，移入 `废弃内容/` 目录（按年月子目录归档），在文件头标注废弃原因+替代路径。废弃内容只增不删不改，永久保留用于溯源。

## Anti-patterns

- 不要在 raw/ 里直接写笔记
- 不要让 wiki/B层/C层 页面变成"孤岛" — 每个页面至少有一个 [[wikilink]]
- 不要产出后不回流 — D层的新认知必须回到B层
- 不要只封装不更新 — 技能用过发现问题要回溯修改B层规则
- **不要直接删除 vault 内任何文件** — 删/替/重构一律移入 `废弃内容/`，保留溯源链

## 这个系统的核心思想

**知识不是存了就完了。** 大多数人用 Obsidian 只是"记笔记"，记完就忘。

ABCD 循环逼你做四件事：
1. **A→B**：看完一篇文章，不是收藏，是提炼出可复用的方法论
2. **B→C**：方法论用过三次，封装成 Claude 一键调用的技能
3. **C→D**：技能驱动产出（写文章、做产品、管项目）
4. **D→A**：产出后的复盘再流回方法论层

这样你的知识库不是"收藏夹"，是**自生长的第二大脑**。

## 怎么用

1. 把上面的目录结构复制到你自己的 Obsidian vault
2. 根目录放这个 CLAUDE.md
3. 项目名换成你自己的
4. 让 Claude 帮你维护——每次产出新认知，Claude 会自动归到对应的层
5. 每周花 10 分钟跑一次 Lint，检查循环有没有在转
